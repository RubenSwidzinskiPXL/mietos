# Code Of Conduct

Be respectful, practical, and safety-minded.

This project welcomes people learning cybersecurity, building defensive tooling, and improving authorized assessment workflows. It does not welcome harassment, doxxing, hate, or attempts to use the project community to target systems without authorization.

When discussing security work:

- Use local labs, toy examples, or authorized systems.
- Do not post real credentials, customer data, private keys, or flags from active rooms.
- Keep disagreement technical and specific.
- Assume contributors are trying to improve the project unless behavior shows otherwise.

Maintainers may remove content or contributions that create avoidable harm, expose sensitive data, or push the project toward unauthorized abuse.
